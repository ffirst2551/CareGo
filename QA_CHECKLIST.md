# CareGo Web App QA Checklist

อัปเดตล่าสุด: 17 พฤษภาคม 2026  
สภาพแวดล้อมที่ใช้ตรวจ: `http://localhost:3000/`  
วัตถุประสงค์: ใช้เป็น checklist ให้ทีม Antigravity, design, และ dev ไล่ตรวจทีละหน้าก่อนส่งงานรอบถัดไป

## วิธีใช้

- ให้ไล่เช็กทีละหน้าใน desktop และ mobile viewport
- ทุกข้อที่ไม่ผ่านให้เปิด ticket หรือผูกกับ bug report ทันที
- ถ้ายังไม่มี backend จริง ให้เช็กอย่างน้อยว่า UI state, route, และ interaction ไม่พัง

## เกณฑ์กลางที่ต้องเช็กทุกหน้า

- [ ] หน้าเปิดได้จริง ไม่ขึ้น `404`
- [ ] ไม่มี blank state ที่ไม่ได้ตั้งใจ
- [ ] ไม่มี element ซ้อนทับกัน
- [ ] ไม่มีข้อความโดนตัด, ล้น, หรืออ่านไม่ออก
- [ ] ไม่มีปุ่มหรือ link ที่กดแล้วไม่เกิดอะไรขึ้นโดยไม่มีเหตุผล
- [ ] CTA หลักของหน้ามองเห็นได้ชัดโดยไม่ต้องเดา
- [ ] ปุ่ม back / close / next / continue ใช้งานได้
- [ ] ชื่อหน้าและ hierarchy ชัดเจน
- [ ] spacing สม่ำเสมอ อ่านง่าย ไม่แน่นเกินไป
- [ ] icon ที่กดได้มี label หรือความหมายชัดเจน
- [ ] route หลังจากกด CTA ถูกต้อง
- [ ] หน้าไม่มี jump หรือ layout shift แปลก ๆ
- [ ] mobile ใช้งานได้ด้วยนิ้ว ไม่ต้องซูม
- [ ] desktop ไม่ดูเป็นจอมือถือแคบลอยกลางจอแบบเสียพื้นที่เกินจำเป็น
- [ ] ไม่มี console error สำคัญที่ทำให้ flow เสี่ยงพัง

## Responsive Checklist

### Mobile

- [ ] ไม่มี horizontal scroll
- [ ] CTA ล่างไม่โดน safe area บัง
- [ ] bottom nav กดง่าย
- [ ] text ไม่เล็กเกินไป
- [ ] card ไม่แน่นเกินไป

### Desktop

- [ ] หน้าไม่ดูลอยเป็น “มือถือแคบกลางจอ” จนเสียพื้นที่มากเกินไป
- [ ] มี layout decision ที่ชัดว่าเป็น web preview หรือ desktop shell
- [ ] spacing รอบ content สมดุล
- [ ] ไม่มีองค์ประกอบที่เล็กเกินไปเมื่ออยู่บนจอกว้าง

## Exit Criteria ก่อนส่งงานรอบถัดไป

- [ ] ไม่มี route สำคัญขึ้น 404
- [ ] booking flow เดินจบจาก home ถึง confirmation ได้
- [ ] first-time user flow เดินจบได้
- [ ] secondary pages เปิดได้ครบ
- [ ] CTA ทุกตัวทำงานตรงความคาดหวัง
- [ ] ไม่มี dead button สำคัญ
- [ ] ไม่มี unlabeled icon control สำคัญ
- [ ] mobile และ desktop ผ่าน sanity check ทั้งคู่
